================================================================================
MISCELLANEOUS UTILITY PLUGINS (Mini.nvim Suite)
================================================================================

This directory contains various mini.nvim plugins that provide essential
editing utilities. Mini.nvim is a collection of minimal, focused plugins
that are lightweight, performant, and well-integrated. These plugins provide
core editing functionality that enhances the basic Neovim experience.

PLUGINS IN THIS DIRECTORY:
---------------------------

1. mini.comment
   Purpose: Comment/uncomment lines and blocks
   Role: Provides easy commenting and uncommenting of code. Supports both
         line comments and block comments. Works with visual selections
         to comment multiple lines at once.
   Why Required: Commenting code is a fundamental editing operation. This
                 plugin makes it easy to toggle comments with a single
                 keystroke, improving editing efficiency.

2. mini.pairs
   Purpose: Auto-pair brackets, quotes, and other delimiters
   Role: Automatically inserts closing brackets, quotes, and other delimiters
         when you type opening ones. Also handles smart pairing and skipping
         over existing pairs.
   Why Required: Reduces typing and prevents mismatched brackets/quotes.
                 Essential for efficient coding, especially in languages with
                 many brackets and quotes.

3. mini.indentscope
   Purpose: Visual indentation guides
   Role: Displays visual guides showing indentation levels in your code.
         Makes it easier to see code structure and identify indentation
         issues. Helps with code readability and navigation.
   Why Required: Improves code readability by making indentation levels
                 visible. Helps identify indentation errors and understand
                 code structure, especially in deeply nested code.

4. mini.icons
   Purpose: Icon utilities
   Role: Provides icon utilities for use in other plugins and configurations.
         Helps with visual consistency across the editor.
   Why Required: Provides consistent icon support for other plugins and
                 custom configurations. Ensures visual consistency.

5. mini.surround
   Purpose: Surround text with brackets, quotes, tags, etc.
   Role: Allows you to easily add, change, or delete surrounding characters
         around text. For example, you can wrap a word in quotes, change
         single quotes to double quotes, or remove surrounding brackets.
   Why Required: Essential for refactoring and editing. Makes it easy to
                 modify surrounding characters without manual editing.
                 Significantly speeds up common editing tasks.

HOW IT FITS IN THE GRAND SCHEME:
---------------------------------

The misc plugins provide fundamental editing utilities:

- mini.comment enables easy code commenting
- mini.pairs provides auto-pairing for faster typing
- mini.indentscope improves code readability
- mini.icons provides icon support
- mini.surround enables efficient text manipulation

Together, these plugins:
- Speed up common editing tasks
- Reduce typing and errors
- Improve code readability
- Provide essential editing utilities

These are the "bread and butter" plugins that you use constantly while
editing. They're lightweight and don't add complexity, but significantly
improve the editing experience.

WHY MINI.NVIM:
--------------

Mini.nvim plugins are chosen because they are:
- Lightweight and fast
- Well-maintained and stable
- Focused on single responsibilities
- Consistent in their API and behavior
- Easy to configure and use

This makes them ideal for providing essential functionality without bloat.

LOADING STRATEGY:
----------------

- Most plugins load lazily (VeryLazy event) to improve startup time
- mini.pairs loads on InsertEnter (when entering insert mode) as it's needed immediately
- This ensures fast startup while maintaining all functionality
